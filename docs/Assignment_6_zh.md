# 第六周

## 簡述學習目標

- **(Open WebUI)「看」資料**：了解資料品質如何影響模型的表現，學習從 Hugging Face 等平台辨識高品質資料集，並能根據需求自行設計資料
- **(Hugging Face)「用」資料**：學習將資料轉成外部知識庫，透過 RAG 讓模型依據真實資料進行回覆
- **「擴充」&「校準」資料**：了解合成資料在資安稀缺樣本下的角色，學習如何根據原來的資料集生出更多資料，並審核 AI 產出的邏輯，建立高品質的輸出

---

## TASK 1 (Achieving)：資料集蒐集與整理

### 作業內容

1. **主題資料收集**
   - 在 Hugging Face 等平台搜尋相同主題資料集，觀察其資料架構與格式：專業資料集多用了哪些你原本沒想到的欄位？這些欄位對模型判斷可能有什麼幫助？
   - 選擇一個資安相關主題（可和上禮拜主題一樣），根據該主題設計你的資料集的架構，並說明 schema 如何幫助模型理解資料
   - 根據設計的欄位，收集、清洗並整理出一份初步的資料集

2. **知識庫應用**
   - 使用 Open WebUI，將你的資料集結構化資料轉成 Markdown 或帶標籤的段落放入知識庫
   - 比較掛載知識庫前與掛載知識庫後針對同一個問題的回覆差異，觀察模型是否能根據真實資料精準回答

### 截圖內容

- 欄位設計說明
- 對比的資料集
- Open WebUI 知識庫設定
- 掛載知識庫前後對比

## TASK 3 (Outstanding)：Synthetic Data

### 作業內容

1. 說明 synthetic data 在資安領域扮演的角色與價值
2. 沿用 Task 1 設計的資料集 schema，寫 prompt 引導 LLM 生出 10–20 筆合成資料，或者用非 LLM 的演算法方式生出合成資料，說明你這樣設計的理由
3. **品質校準與分析**：從生成的資料中選出 **1 筆最合理的**與 **1 筆不合理的**（AI 瞎掰的，或者合成演算法的瑕疵），並說明你將如何解決這個問題

### 截圖內容

- 合成資料生成指令或演算法截圖
- 合成資料結果截圖
- 品質校準對比分析截圖
